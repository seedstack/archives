Web-Services Transports

  • In Add-ons

The WS add-on supports HTTP and JMS transports. Each URL syntax described below. Standalone HTTP In a standalone environment you need to specify the full URL with the binding address and the port: http://localhost:4578/ws/hello Web server HTTP In a Web server environment you only need to specify the URL pattern: /ws/hello JMS JMS URIs are unchanged whatever the environment.
